# MPC 调试与可视化分析指南

本指南用于指导如何在 ROS 环境下（实飞或播放 rosbag 时）分析 MPC 内部状态，排查无人机悬停不稳、轨迹跟踪发散等问题。

## 1. 话题(Topic)速查表

| Topic 名称 | 数据类型 | 含义 | 最佳可视化工具 |
| :--- | :--- | :--- | :--- |
| `/odom` | `nav_msgs/Odometry` | 经过滤波的当前真实状态 | RViz / PlotJuggler |
| `/mpc_debug/mode` | `std_msgs/Int8` | 状态机 (0:起飞, 1:悬停, 2:跟踪) | PlotJuggler |
| `/mpc_debug/ref_pose` | `geometry_msgs/PoseStamped` | MPC **当前(Step 0)** 参考位姿 | PlotJuggler |
| `/mpc_debug/raw_control`| `std_msgs/Float32MultiArray`| MPC **当前(Step 0)** 输出控制量 `[T, wx, wy, wz]` | PlotJuggler |
| `/mpc_debug/acado_x0` | `std_msgs/Float32MultiArray`| 求解器接收到的**初始状态反馈** | PlotJuggler |
| `/mpc_debug/acado_ref_path` | `nav_msgs/Path` | 求解器视野内的 **未来 N 步参考轨迹** | RViz |
| `/mpc_debug/acado_pred_path`| `nav_msgs/Path` | 求解器计算出的 **未来 N 步预测轨迹** | RViz |

---

## 2. 空间域诊断：使用 RViz 看轨迹预测

**适用场景**：排查 MPC 预测是否发散、预测步长 (Horizon) 内的轨迹是否平滑、是否因为权重设置导致轨迹突变。

**操作步骤**：
1. 打开终端运行 `rviz`。
2. 修改 `Fixed Frame` 为 `world` 或 `map`。
3. 点击 `Add` -> `By topic`，添加以下内容：
   - `Odometry` -> `/odom` (建议把 Shape 设为 Axes 看飞机真实姿态)
   - `Path` -> `/mpc_debug/acado_ref_path` (颜色设为 **绿色**，代表期望轨迹)
   - `Path` -> `/mpc_debug/acado_pred_path` (颜色设为 **红色**，代表预测轨迹)

**现象分析**：
* ? **正常悬停**：绿色是一坨紧密的点，红色也是一坨紧密的点，且红绿重合，包围在 `/odom` 周围。
* ? **预测发散 (弹簧效应)**：红色轨迹像弹簧一样疯狂甩动或伸长。
  * *原因*：惩罚权重（如 `cost_vx`, `cost_wx`）设置不合理，或者预测步长 $N$ 过大而控制频率过低。
* ? **稳态误差**：红色和绿色轨迹稳定，但偏离了 `/odom` 真实位置。
  * *原因*：通常是 `hover_thrust` 估计不准，导致 MPC 认为给一点油门就能悬停，但实际飞机在掉高；或存在未建模的外部扰动（如风偏、重心偏移）。

## 3. 时间域诊断：使用 PlotJuggler 看曲线

**适用场景**：精细化排查时延、控制量饱和、状态跳变等问题。

**操作步骤**：
1. 启动工具：`rosrun plotjuggler plotjuggler`
2. 导入当前正在播放的 ROSBag 或订阅实时 Topic。
3. 拖拽建立以下视图对比：

### 视图 A：位置跟踪误差对比
* 将 `/odom/pose/pose/position/x` 和 `/mpc_debug/ref_pose/pose/position/x` 拖入同一个窗口。
* **排查点**：如果 `odom` 曲线始终比 `ref` 曲线慢半拍，或者有固定偏差，检查 `hover_thrust` 和积分项/控制延时。如果 `odom` 出现剧烈毛刺，说明 VRPN 数据丢包或动捕刚体遮挡。

### 视图 B：控制量饱和分析
* 展开 `/mpc_debug/raw_control/data`。
* `data[0]` 是推力 Thrust，`data[1..3]` 是姿态角速度 wx, wy, wz。
* **排查点**：观察 `data[0]` 是否经常顶到你设置的上限（`T_max`）或下限（`T_min`）。如果长期处于饱和状态，MPC 将失去调节能力，飞机一定会失控。

### 视图 C：状态机打断监控
* 将 `/mpc_debug/mode/data` 单独放一个图。
* **排查点**：观察飞机失控瞬间，状态机是否意外从 1(Hover) 掉到了 0(Takeoff) 或在 1 和 2 之间反复横跳。
